# Reporting redesign prototypes

Open each HTML file directly in a browser. Navigation controls switch between
the mocked pages without a server.

- `prototype-a-financial-hub.html`: period-centered financial reporting with
  separate pages for periods, payables, disbursements, tax, and data quality.
- `prototype-b-action-center.html`: task-first home page with operational queues
  and a global giving ledger.
- `prototype-c-ledger-first.html`: recipient/ledger-centered model optimized for
  frequent payment and reconciliation work.
- `prototype-d-recommended-hybrid.html`: refined recommended direction combining
  the financial structure of A with the payment-first action model of B.
- `prototype-e-full-site.html`: connected full-site direction covering the
  merchant-admin structure, key product-cost workflows, reusable modal patterns,
  and customer-facing storefront surfaces.
- `site-interface-inventory.md`: complete page, detail-view, modal, and
  supporting-surface inventory with confirmed design decisions.
- `prototype-e-gap-audit.md`: route-by-route and storefront-block comparison of
  current functionality against Prototype E, including missing, partial, and
  intentionally consolidated behavior.

All examples use static illustrative data. They are information-architecture
prototypes, not proposed production styling or finalized copy.

Use `plain-language-copy-guide.md` for financial labels and help-text patterns
when refining prototypes or implementing production pages.
