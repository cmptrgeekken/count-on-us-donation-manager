# Documentation Archive

This folder keeps historical planning and backlog material available without making it prominent.

Archived docs are useful for context, but they are not the active work queue. For outstanding work, use GitHub Issues.

## Folders

- `backlog/` — local backlog material that predates the GitHub-Issues-as-source-of-truth policy.
- `plans/` — phase plans, issue plans, audits, and review logs that are historical or superseded by GitHub Issues.

If an archived document still matters to current work, link it from the relevant GitHub issue rather than moving it back into the prominent docs set.
